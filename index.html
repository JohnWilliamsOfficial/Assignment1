Project Idea

Overview

My project idea is an application that can be used to authenticate game & trading cards such as Yu-Gi-OH, Pokemon, Baseball and more to ensure that they are real. At a very basic level, a camera with a database of particular trading card branded images of authentic and fake cards used to distinguish the fake would be the goal, ultimately this could expand to whole machinery that can use this system.
This application can be pitched to reselling companies such as Troll and Toad & Mighty Collectibles to make it as seamless as possible when verifying trading cards. I would like this application to be used commercially first but later to be rolled out to the wider consumer base after a certain period.

Motivation

A recent article written in 2022 by the Nintendo Wire has advised that over 43 billion cards have been sold by Pokemon in March 22 alone (1). The line is blurred when it comes to what’s real and fake, utlising a simple phone camera to scan the front & back of a card to determine its authenticity would be an absolute game changer for this problem. This not only lies with Pokemon cards but for baseball cards and Yu-Gi-Oh and much more. The market is so large and there are many different opportunities for companies to work without even discussing the consumer space at the moment.

Description

Inspired by the market and made for the consumer, this is how I would like to describe my product. Not only are we able to penetrate such a huge market space but it’d be ever expanding for any other product that we’re able to set our eyes on, with so many different companies that create(d) trading cards, the sky's the limit with how this app can be utlised.

The app in its final version would have different options for the type of company that it’s for, EG. Pokemon, Magic the Gathering etc. We’d implement an ever growing database of images that can sort for discrepancies between certain cards and be able to give a stamp/verification seal which would allow reselling to be easier. Imagine StockX with shoes but for cards - We’d simplify the process process to allow for growth in the community.

Tools and Technologies

Some tools required at a very basic level would be:

  • A computer to write the software and access the storage/database
  • A phone with a high quality camera to test out the application
  • A storage system that will hold the database of fake/real trading card images that allows for distinguishing of discrepancies with the cards
  • A virtual machine/raspberry PI as a second system for troubleshooting

Skills Required

The skills required to complete this project would be

  • CSS Knowledge to create a phone application
  • Code writing software such as Adobe Dreamweaver which will give you access to Java, HTML, CSS & more for the software to start being written.
  • Possible AI integration for the app to get smarter/learn over time.

Desired Outcome

If the project is successful, the best positive outcome is that the consumers should be able to use their mobile phones to start authenticating their trading cards. This would be such a huge win as they would be able to differentiate fake and real cards at ease, eliminating most of the fake resellers in the market from the equation.
Ultimately I want the app to be able to have multiple different company branded cards and an easy to User Interface that allows the individual to search through ease.
